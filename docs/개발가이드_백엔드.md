## 3. 도메인 모듈 패턴

### 3.1 표준 디렉토리 구조

```
domain/{domain-name}/
├── controller/
│   └── {Domain}Controller.java  ← REST API 엔드포인트
├── dto/
│   ├── {Domain}Request.java     ← 요청 DTO
│   └── {Domain}Response.java    ← 응답 DTO
├── entity/
│   └── {Domain}.java            ← JPA Entity
├── repository/
│   └── {Domain}Repository.java  ← Spring Data JPA
└── service/
    └── {Domain}Service.java     ← 비즈니스 로직
```

### 3.2 레이어 책임 분리

| 레이어 | 책임 | 규칙 |
|--------|------|------|
| **Controller** | HTTP 요청/응답, 유효성 검사 | 비즈니스 로직 금지, Service 호출만 |
| **Service** | 비즈니스 로직, 트랜잭션 | `@Transactional` 사용, Repository 호출 |
| **Repository** | 데이터 접근 | JPA 쿼리 메서드, `@Query` |
| **Entity** | DB 테이블 매핑 | `@Entity`, `@Table` |
| **DTO** | 요청/응답 데이터 | Entity 직접 노출 금지 |

### 3.3 코드 패턴

```java
// Controller
@RestController
@RequestMapping("/api/v1/models/formulas")
@RequiredArgsConstructor
public class CostModelController {

    private final CostModelService service;

    @GetMapping
    public ApiResponse<List<CostFormulaResponse>> list() {
        return ApiResponse.success(service.findAll());
    }

    @PostMapping
    @PreAuthorize("hasRole('ADMIN')")
    public ApiResponse<CostFormulaResponse> create(@Valid @RequestBody CostFormulaRequest req) {
        return ApiResponse.success(service.create(req));
    }
}

// Service
@Service
@Profile("!mock")
@RequiredArgsConstructor
@Transactional(readOnly = true)
public class CostModelService {

    private final CostFormulaRepository repository;

    public List<CostFormulaResponse> findAll() {
        return repository.findAll().stream()
            .map(CostFormulaResponse::from)
            .toList();
    }

    @Transactional
    public CostFormulaResponse create(CostFormulaRequest req) {
        CostFormula formula = CostFormula.builder()
            .name(req.getName())
            .expression(req.getExpression())
            .build();
        return CostFormulaResponse.from(repository.save(formula));
    }
}

// Entity
@Entity
@Table(name = "cost_formulas")
@Getter @NoArgsConstructor(access = AccessLevel.PROTECTED)
@Builder @AllArgsConstructor
public class CostFormula {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String name;

    private String expression;

    @JdbcTypeCode(SqlTypes.JSON)
    @Column(columnDefinition = "jsonb")
    private List<String> departments;
}

// DTO (Response)
@Getter @Builder
public class CostFormulaResponse {
    private Long id;
    private String name;
    private String expression;
    private List<String> departments;

    public static CostFormulaResponse from(CostFormula entity) {
        return CostFormulaResponse.builder()
            .id(entity.getId())
            .name(entity.getName())
            .expression(entity.getExpression())
            .departments(entity.getDepartments())
            .build();
    }
}
```

## 7. 인증/보안

### 7.1 JWT 토큰 흐름

```
1. POST /api/v1/auth/login
   요청: { employeeId: "admin1", password: "password" }
   응답: { token: "eyJ...", refreshToken: "eyJ...", user: {...} }

2. API 요청
   Header: Authorization: Bearer eyJ...
   → JwtAuthenticationFilter → SecurityContext

3. POST /api/v1/auth/refresh
   요청: { refreshToken: "eyJ..." }
   응답: { token: "eyJ...(새 토큰)" }

4. POST /api/v1/auth/logout
   → Redis 블랙리스트에 토큰 등록
```

### 7.2 계정 잠금

```
5회 연속 로그인 실패 → Redis에 실패 카운트 저장
TTL: 30분 후 자동 해제
관리자 수동 해제 가능

## 12. 피드백 Excel 생성 (Apache POI)

### 12.1 개요

견적서 분석 중 이상치가 발견되면, **원본 Excel 파일에 노트(Comment)와 셀 하이라이트를 추가**하여 협력사에 피드백합니다.

```
프론트엔드 → GET /api/v1/quotations/{id}/analysis/feedback-excel
           → 백엔드 (FeedbackExcelService)
           → NAS에서 원본 파일 로드 (Apache POI XSSFWorkbook)
           → 이상치 셀에 노트 + 하이라이트 추가
           → 원본 양식/수식/도형/외부링크 100% 보존
           → 피드백 Excel 바이트 배열 반환
           → 프론트엔드에서 파일 다운로드
```

### 12.2 왜 Apache POI만 사용하는가?

| 라이브러리 | 문제점 |
|-----------|--------|
| **openpyxl (Python)** | 외부 링크(externalLink), 도형(drawing) 손상 → "읽을 수 없는 내용" 오류 |
| **SheetJS (JavaScript)** | 셀 스타일/수식을 재생성하여 원본 양식 깨짐 |
| **ExcelJS (JavaScript)** | 파일 구조를 재생성하여 원본 손상 |
| **Apache POI (Java)** | ✅ 원본 ZIP 구조를 그대로 유지하며 수정/추가만 처리 |

### 12.3 원본 보존 원칙

```
❌ 금지: 원본 데이터를 직접 수정
✅ 허용: 노트(Comment) 추가 + 셀 배경색 변경만

이유:
1. 견적서는 협력사가 작성한 공식 제출 문서 (법적 증거)
2. 원본 수정 시 감사 추적 불가 + 책임 소재 불명확
3. 원본 수식/외부 참조가 깨지면 데이터 정합성 훼손
4. 노트/하이라이트만 추가 → 협력사가 직접 확인/수정 → 수정본 재제출
```

    // 피드백 Excel 생성
    public byte[] generateFeedbackExcel(String originalFilePath,
                                         List<AnomalyFeedback> anomalies) throws IOException {
        try (XSSFWorkbook workbook = new XSSFWorkbook(new FileInputStream(originalFilePath))) {
            XSSFSheet sheet = workbook.getSheetAt(0);
            XSSFDrawing drawing = sheet.createDrawingPatriarch();

            for (AnomalyFeedback anomaly : anomalies) {
                // 1. 셀 노트(Comment) 추가 — 리치 텍스트 (제목: 빨간 볼드)
                Comment comment = drawing.createCellComment(anchor);
                comment.setString(richTextString);
                comment.setAuthor("원가분석시스템");
                cell.setCellComment(comment);

                // 2. 셀 하이라이트 — 기존 스타일 보존 + 노란 배경만 추가
                XSSFCellStyle merged = workbook.createCellStyle();
                merged.cloneStyleFrom(cell.getCellStyle()); // 기존 테두리/수식 유지
                merged.setFillForegroundColor(yellow);
                cell.setCellStyle(merged);
            }

            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            workbook.write(bos);
            return bos.toByteArray();
        }
    }
}
```
